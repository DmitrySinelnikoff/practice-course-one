# Рассмотрите и опишите, какие языки программирования необходимо знать для работы в сфере веб-разработки?

## Ответ

Для веб-разработки требуется следующий набор технологий:

1. Обязательно HTML и CSS (хоть это и не языки программирования, но необходимо упомянуть). Этот набор позволит на минимальном уровне собрать как минимум статическую страницу, которая будет показана с определенной стилизацией.
2. Любой серверный язык программирования. Он необходим для реализации непосредственно логики работы сайта: авторизация, хранение, получение информации, проведение каких-то операций, которые либо невозможно произвести на клиенте, либо нецелесообразно. Примеры таких языков: PHP, GO, Java, C#.
3. Клиентский язык программирования (JavaScript). Нужен для обработки логики на стороне клиента (в браузере). Позволяет оживить страницу путем добавления интерактивных элементов и также позволяет делать HTTP-запросы, что может помочь с более интуитивным показом контента.

- HTML — язык гипертекстовой разметки, который является стандартом веб-разработки и позволяет делать макет-каркас.
- CSS — таблицы стилей, которые задают внешний вид для HTML-разметки.
- PHP (опишу как пример серверного языка) — интерпретируемый язык общего назначения с динамической типизацией.
- JavaScript — это интерпретируемый мультипарадигменный язык программирования, который чаще всего используется для придания интерактивности веб-страницам. Фактически является стандартом в веб-разработке.

# Какие принципы и паттерны программирования широко используются при создании веб-приложений?

## Ответ

Есть определенный набор принципов, которые в целом используются во всех сферах при написании кода:

- SOLID — набор принципов объектно-ориентированного проектирования. Помогает делать код более расширяемым, тестируемым и менее связанным.
- DRY (Don't Repeat Yourself) — избегать дублирования логики.
- KISS (Keep It Simple, Stupid) — не усложнять решение без необходимости.
- YAGNI (You Aren't Gonna Need It) — не реализовывать функциональность «на будущее», пока она действительно не нужна.

Если говорить про паттерны построения самого веб-приложения, то вот основные виды:

- MVC (Model-View-Controller) — один из самых распространенных паттернов в веб-разработке. Разделяет модель данных, пользовательский интерфейс и обработку запросов.
- Layered Architecture — разделение приложения на слои (контроллеры, сервисы, репозитории, инфраструктура и т.д.).

Если говорить про паттерны порождающего программирования, которые чаще используются в веб-разработке, то это:

- Repository — инкапсулирует работу с хранилищем данных. Допустим, репозиторий, который будет обеспечивать получение полного списка товаров на сайте.
- Strategy — позволяет выбирать алгоритм во время выполнения. Позволяет фактически прятать за этой стратегией разные реализации одного механизма, допустим, разные виды хранения кеша.
- Observer — используется для событий и подписок. Используется для реализации событийной системы. Пример: когда создается пользователь, то пишется лог в файл по событию.
- Singleton — применяется реже, но иногда используется для объектов с единственным экземпляром. Пример: получение подключения к базе данных, в таком случае мы гарантируем, что оно будет только одно.
